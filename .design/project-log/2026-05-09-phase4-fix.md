# Project Log - 2026-05-09 - Phase 4 Fixes

## Task Overview
Fixed Phase 4 regressions in Hub handlers related to GCP Service Account minting and Broker inbound message topic parsing.

## Changes

### 1. GCP Service Account Minting Fix
In `pkg/hub/handlers_gcp_identity.go`, the `mintGCPServiceAccount` function was incorrectly using the Fabric `projectID` as the GCP `ProjectID` when creating the `store.GCPServiceAccount` record. This was changed to use `hubGCPProjectID` (which comes from `s.config.GCPProjectID`), as the service account is actually created in the Hub's managed GCP project, not necessarily a project with an ID matching the Fabric project ID.

### 2. Broker Inbound Topic Parsing Fix
In `pkg/hub/handlers_broker_inbound.go`, updated the `parseAgentMessageTopic` function and associated comments/error messages to use `project` instead of `grove` in the topic string.
- Topic format updated to: `fabric.project.<projectID>.agent.<agentSlug>.messages`
- Renamed internal variables from `groveID` to `projectID` for consistency with the new terminology.
